# Project Stack Guide

Consolidated from `PROJECT-STACK.md` on 2026-04-07.

## Flow Finance + Clinic Automation

This is the primary project context.

### Main Skills / Plugins

- `GSD`
- `Superpowers`
- `context7`
- `n8n-mcp`
- `UI/UX Pro Max`

### Auxiliary Skills / Plugins

- `marketing-skills`
- `gstack`
- `obsidian-skills`

### Practical Usage

- `GSD`: scope, phases, backlog, v1/v2, execution by stage
- `Superpowers`: brainstorming, technical plan support, debugging, review, verification
- `context7`: docs for Firebase, n8n, Vercel, auth, APIs, and touched dependencies
- `n8n-mcp`: clinic automations and workflow review
- `UI/UX Pro Max`: app screens, dashboard, forms, practical UX clarity
- `marketing-skills`: later for positioning, landing, and messaging
- `gstack`: punctual QA, investigate, benchmark, review, or release support
- `obsidian-skills`: external documentation of flows, decisions, and backlog if needed

### Rules

- treat Flow Finance plus clinic automation as one ecosystem
- do not use `gstack` as a competing primary workflow alongside GSD
- do not over-index on design before the main product flow works
- critical automation requires human confirmation where operational risk exists

## Rule Of Thumb By Category

### Engineering and product execution

- `GSD + Superpowers + context7`

### Operational automation

- `n8n-mcp + context7 + GSD`

### UI and interface refinement

- `UI/UX Pro Max`

### Marketing and monetization

- `marketing-skills`

### Knowledge capture and organization

- `obsidian-skills`

### Punctual QA, release, and investigation

- `gstack`

## Mother Rule

Choose one main workflow per project:

- main technical project -> `GSD` as backbone
- technical quality support -> `Superpowers`
- docs lookup -> `context7`
- automation -> `n8n-mcp`
- marketing -> `marketing-skills`
- knowledge organization -> `obsidian-skills`
- punctual QA/review/release -> `gstack`
