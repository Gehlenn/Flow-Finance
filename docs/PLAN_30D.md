# 30-Day Operating Plan

Consolidated from `PLAN-30D.md` on 2026-04-07.

## 30-Day Objective

For this repository, interpret the 30-day plan only through the Flow Finance lens:

- Flow Finance plus clinic is the only active project track here
- parallel efforts such as channel, ebook, and games are out of scope for this repository context

## Guiding Principles

- focus on few fronts
- prioritize real usage and validation
- prioritize monetization before expansion
- prefer ugly but functional progress over attractive stagnation
- work should fall into one of:
  - product
  - distribution
  - monetization
  - useful documentation

## Main Outcome Targets

### Flow Finance + clinic

- principal flow mapped
- central automation designed or partially functional
- human confirmation rules defined
- path defined between service, schedule, and finance
- v1 backlog prioritized

## Weekly Bias

### Week 1: clarity and structure

For Flow Finance + clinic:

- map the full clinic flow
- list what already exists versus what is fragmented
- define mandatory v1 scope
- define what stays out for now

### Week 2: build the core

For Flow Finance + clinic:

- design or revise the main n8n automation
- define critical messages and human confirmation points
- choose minimum data model
- evaluate Firebase to Supabase transition or temporary coexistence
- identify operational and technical risks

### Week 3: first practical validation

For Flow Finance + clinic:

- test with real or semi-real scenarios
- identify breaks in understanding, scheduling, authorization, record capture, and finance integration
- fix the worst bottlenecks
- refine v1 backlog

### Week 4: consolidate and decide

For Flow Finance + clinic:

- close a minimum usable version of the main flow
- document what works and what is still manual
- define the next 30 days from real usage evidence

## Focus Rules For The Month

Do:

- work on the main project almost every day
- document important decisions
- review priorities weekly

Do not:

- open new fronts without need
- over-invest in visuals before the core works
- confuse movement with progress

## Stack Bias For This Plan

### Flow Finance + clinic

- `GSD` for planning and phases
- `Superpowers` for execution, debugging, and review
- `context7` for stack and docs
- `n8n-mcp` for automation
- `UI/UX Pro Max` after the core is working

## Success Metric

At the end of 30 days, evaluate:

1. Did the clinic project get closer to real use?
2. Did Flow Finance become clearer as a product?
3. Is there at least one plausible short-term monetization path inside the product direction?
4. Did the core flow become more operationally credible?
5. Was there less dispersion than in the prior month?

If 4 of 5 are yes, the month was good.
