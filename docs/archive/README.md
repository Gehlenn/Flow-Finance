# Arquivo de Documentação (`docs/archive/`)

Este diretório reúne documentos históricos de versões anteriores para reduzir ruído no fluxo principal de leitura.

Regra de ouro: **nada aqui é fonte de verdade operacional viva**. Se um ponto voltar a orientar operação atual, promova a informação relevante para um documento vivo em `docs/` e mantenha o original como memória histórica.

## Critério

- arquivos de releases antigas podem ser movidos para este diretório
- sumários de PR, checkpoints antigos, prompts de go/no-go e diagnósticos técnicos encerrados também pertencem aqui
- auditorias datadas que não representam o status vivo ficam aqui
- o historico permanece preservado no Git
- documentação ativa deve permanecer em `docs/` e na raiz quando aplicável
