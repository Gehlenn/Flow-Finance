# Flow Finance Product Plan

Consolidated from `FLOW-FINANCE-PRODUCT-PLAN.md` on 2026-04-07.

## Positioning

**Flow Finance is an intelligent cash-flow app for service businesses, connected to real operations.**

## Product Value

The product should not behave like a generic finance app or an AI showcase. Its practical differentiation is the link between:

- service operations
- scheduling and attendance context
- projected revenue
- confirmed revenue
- cash visibility

## Primary Persona

- autonomous professionals
- small service businesses
- clinics, consultories, aesthetics, appointment-driven businesses

Priority validation case:

- dental clinic

## What The Product Is Not Right Now

- not a broad finance super-app
- not an Open Finance platform
- not a magical autonomous CFO
- not a system that must solve all finance jobs on day one

## Short-Term Product North

Validate real usage around this flow:

- attendance
- scheduling
- human confirmation
- operational record
- projected revenue
- confirmed revenue
- financial visibility

## Priority Filter

A feature is not a current priority unless it improves at least one of:

1. cash-flow control
2. linkage between operations and finance
3. decision clarity
4. real clinic usage
5. short-term validation or monetization

## Product Core

Keep these in focus:

- cash flow
- useful transaction records
- projected and realized revenue
- operational linkage
- consultative AI
- clear dashboard

## Freeze Or Remove From Main UI

Freeze or de-emphasize:

- Open Finance / Pluggy
- OCR as a central theme
- advanced simulations and sophisticated automation
- oversized autonomous-AI promises

Frontend implication:

- hide, remove, or strongly deprioritize any UI that suggests frozen scope is part of the current core product

## Backlog Shape

### Now

- simplify product scope and messaging
- remove Open Finance / Pluggy from main UI
- simplify navigation
- strengthen financial core UX
- model projected and realized revenue clearly
- align AI to consultative support

### Next

- deepen clinic-to-finance linkage
- make dashboard more decision-oriented
- shape the product toward service-business language and workflows

### Later

- OCR
- more advanced automation
- broader vertical abstraction

## Coding Guidance

For engineering work in the current cycle:

1. remove Open Finance / Pluggy from the primary interface
2. reduce distractions unrelated to the core use case
3. reinforce cash flow, transactions, dashboard, projected/realized revenue, and consultative AI
4. preserve frozen code if it can be reused later, but keep it out of the main experience
5. align UI copy and structure to the current product thesis
